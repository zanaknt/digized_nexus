# Digized Nexus Workflow Rules

This repository follows a lightweight, controlled workflow for Digized Nexus development.

- Keep changes minimal and focused.
- Do not add unnecessary files.
- After completing each feature step, wait for user review.
- If the user approves:
  - check git status
  - stage relevant changes
  - create a clear commit message
  - push to origin main
- Never use force push.
- Never delete files unless explicitly requested.
- Never push unfinished or broken work.
- Prefer one clean commit per meaningful feature step.
